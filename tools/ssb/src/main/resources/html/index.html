SORCER Service Browser (SSB)

The SORCER service browser comes with support for the following features

  • Service Admin
  • ServiceUI support
  • Service icon support
  • Unicast & Multicast discovery
  • Filters for interfaces, groups, Name attributes & lookup locators
  • Hierarchical & graphical views
  • Displays Lookup service events
  • Create multiple windows with different filters
  • Displays service codebase URLs
  • Displays service codebase JAR contents
  • Multicast request & multicast announcement monitor

Tree views

  • Lookup services
    Provides information on all the services that are registered with each Lookup Service.
  • Services
    Shows a list of unique services.

By expanding each service node you< can see the following information

  • ServiceID
  • Codebase URLs
  • Interfaces & methods
  • Lookup attributes
  • Service methods

Graphical view

By selecting a Lookup Service node in the tree this view will change to a graphical representation of the Lookup Service and the services registered with it. By moving the cursor over each service icon a popup window will display that service's full name and the lookup attributes in the properties window.

To see all a service's attributes in the popup layer, use the View menu to enable this feature

Double click on a service glyph to take you to the service admin view

Service UIs

If you select a service in the tree view that supports a JComponent service UI then the service UI will be displayed in this window.

If your service supports other JFC factories then you can use the "Windows" menu to launch JFrame, JDialog & JWindow serviceUIs

By default the browser will display the ServiceUI's role name e.g MainUI as the title. If you set the AccessibleContext name on your JComponent the browser will use this name instead of the role name e.g. MyName

Service icons

If your service has a ServiceType attribute that return an Image from ServiceType.getIcon() then the browser will attempt to display this icon in the service's tree

Multiple windows

To open a new browser window use the File->New Window... menu option

The Window menu has options for cascade/tile plus a list of open windows to select from

Using filters

There are four types of filters you can apply to the lookup disovery process.

These can be set via the Filters menu.
You can use C++ style comments in each of the filter views to comment out values

  1. Interfaces
    Type in the full name of the service's interface classes that you want to filter on.
    For example net.jini.event.EventMailbox

  2. Groups
    These filters apply the "initialMemberGroups" set by a Lookup Service.
    There are two special cases to keep in mind.
    For the public group, enter "PUBLIC", for all groups enter "ALL_GROUPS"

  3. LookupLocators
    If you specify any LookupLocators then the brower will only perform Unicast Discovery
    Lookup locators are specified as Jini URLs in the format
    jini://<host>:<port>
    You can leave off the port value to specify the default Lookup Service port.
  4. Names
    This filter allows you to filter on a service's Name attribute
    Please note that this filter overrides the interfaces filter
  5. Plugin filters
    This filter allow you to programatically filter out services that you don't want the browser to display

To reset the filters, use the Filters->Reset filters menu

Unicast Discovery

You can add additional Lookup Services that are running outside of the default multicast range by using the "Options->Unicast discovery" menu option.

Enter the jini URL of the Lookup Service in the format jini://<host>:<port> If the LUS is using the default port then you do not have to specify the port

Log file settings

The browser creates a log file called browserLog.txt in the working directory

To monitor the log file use the "View->Log file" menu option

You can change the name and location of the log file by specify the full path in the system property -Dssb.logFile=

The maximum size (in KB) of the log file can be set via the system property -Dssb.logLen= the default size before the log file is overwritten is 500KB

Exported host name or IP Address

You can override the default host name exported by the browser's LUS listener by adding the following JVM command to the browser's start script

-Dssb.browser.exportHost
